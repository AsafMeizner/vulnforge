# VulnForge

AI-powered vulnerability research platform. Find, triage, and report security issues in open-source projects ‚Äî with an autonomous multi-agent pipeline, MCP integration, and a plugin system.

## Quick Start

```bash
# Install dependencies
npm install

# Seed the database with built-in tools and sample data
npm run seed

# Start both frontend (Vite :5173) and backend (Express :3001) in watch mode
npm run dev
```

Open http://localhost:5173 in your browser.

## Features

### Core Platform
- **Findings database** ‚Äî SQLite-backed vulnerability store with severity, CVSS, CWE, code snippets, and full audit trail
- **Projects** ‚Äî Import local paths or remote repos; track scan history and vuln counts per project
- **Scanner** ‚Äî Run any combination of the 13 built-in static analysis tools against a project; live terminal output via WebSocket; per-tool progress tracking
- **Checklists** ‚Äî OWASP and custom security checklists, linked directly to findings

### AI Features
- **AI Triage** ‚Äî Structured JSON triage (severity, exploitability, false-positive risk, CVSS, CWE, tier A/B/C) via any configured provider
- **Report Generation** ‚Äî Four report types generated by AI:
  - **Full Disclosure** ‚Äî email or GitHub Advisory format matching the security-solver template
  - **Email Body** ‚Äî plain-text private disclosure email
  - **GitHub Advisory** ‚Äî structured for the GitHub Security Advisory form
  - **Executive Summary** ‚Äî plain-language briefing for non-technical stakeholders
- **AI Agent** ‚Äî Autonomous multi-step agent (`POST /api/ai/agent`) that breaks down a research goal into tool calls and executes them
- **AI Chat** ‚Äî Conversational interface on the AI page

### Integration
- **MCP Server** ‚Äî Model Context Protocol server at `/mcp` (SSE transport); exposes all findings, projects, and scan control as MCP tools
- **Plugin System** ‚Äî Load external scanners or reporters via the Plugins page
- **WebSocket** ‚Äî Real-time scan output, completion events, and triage results pushed to the UI

### Data flow

```
User action
    ‚Üí React component (src/pages/)
    ‚Üí api.ts fetch wrapper
    ‚Üí Express route (server/routes/)
    ‚Üí db.ts CRUD helper (sql.js SQLite)
    ‚Üí JSON response

Scan execution
    ‚Üí POST /api/scans  { project_id, tools[], auto_triage }
    ‚Üí ScanQueue enqueue
    ‚Üí Tool subprocess (Python script in /tools/)
    ‚Üí stdout lines ‚Üí WebSocket broadcast ‚Üí React terminal
    ‚Üí Findings parsed ‚Üí written to DB
    ‚Üí auto_triage ‚Üí AI pipeline ‚Üí triage:complete WS event ‚Üí toast

AI triage
    ‚Üí POST /api/ai/triage/:id  (202 Accepted)
    ‚Üí pipeline.triageFinding()
    ‚Üí routeAI() ‚Üí active provider (OpenAI / Claude / Ollama)
    ‚Üí parseTriageResponse() ‚Üí JSON
    ‚Üí updateVulnerability(ai_triage, ai_summary, cvss, cwe)
    ‚Üí WS triage:complete broadcast
```

## MCP Integration

VulnForge exposes an MCP (Model Context Protocol) server at `http://localhost:3001/mcp` using SSE transport. This lets any MCP-compatible AI assistant (Claude Desktop, Cursor, etc.) query and control the platform.

### Connecting Claude Desktop

Add to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "vulnforge": {
      "url": "http://localhost:3001/mcp"
    }
  }
}
```

### Available MCP tools

The MCP server exposes the same capabilities as the REST API as callable tools, including listing findings, starting scans, running triage, and generating reports.

## Plugin System

Plugins extend VulnForge with custom scanners, reporters, or integrations.

### Plugin manifest (`plugin.json`)

```json
{
  "name": "my-scanner",
  "version": "1.0.0",
  "type": "scanner",
  "description": "Custom pattern scanner",
  "entry": "scanner.py",
  "config_schema": {
    "pattern": { "type": "string", "description": "Grep pattern" }
  }
}
```

### Plugin types

| Type | Description |
|------|-------------|
| `scanner` | Python script that outputs findings as JSON to stdout |
| `reporter` | Formats findings for external submission |
| `integration` | Connects to external services (Jira, Slack, etc.) |

### Installing a plugin

Via the UI: Plugins page ‚Üí paste a GitHub URL or local path.

Via API:
```bash
curl -X POST http://localhost:3001/api/plugins \
  -H 'Content-Type: application/json' \
  -d '{"source_url": "https://github.com/org/my-scanner"}'
```

## AI Providers

Configure providers on the Settings page. Only one provider is active at a time.

| Provider | Models | Notes |
|----------|--------|-------|
| OpenAI | gpt-4o, gpt-4o-mini, o1, o3 | Requires `OPENAI_API_KEY` |
| Claude (Anthropic) | claude-sonnet-4, claude-opus-4 | Requires `ANTHROPIC_API_KEY` |
| Ollama | llama3.2, qwen2.5, mistral, ‚Ä¶ | Local inference, no key needed |

Set keys either in Settings UI or via environment variables ‚Äî the UI stores them in the local SQLite database.

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/stats` | Platform statistics |
| GET/POST | `/api/vulnerabilities` | List / create findings |
| GET/PUT/DELETE | `/api/vulnerabilities/:id` | Single finding |
| GET/POST | `/api/projects` | List / import projects |
| GET/POST | `/api/scans` | List / start scans |
| POST | `/api/scans/profile` | Start scan with a named profile |
| GET | `/api/tools` | List analysis tools |
| POST | `/api/ai/triage/:id` | Start AI triage for a finding (202) |
| POST | `/api/ai/chat` | Conversational AI |
| POST | `/api/ai/agent` | Run autonomous agent with a goal |
| POST | `/api/reports/generate` | Generate a report (`disclosure`/`email`/`advisory`/`summary`) |
| GET | `/api/reports` | List generated reports |
| GET/PUT | `/api/ai/providers` | AI provider configuration |
| GET | `/api/health` | Health check |

WebSocket: `ws://localhost:3001/ws` ‚Äî receives `scan:start`, `scan:output`, `scan:complete`, `scan:error`, `triage:complete`, `triage:error`, `queue:drain` events.
